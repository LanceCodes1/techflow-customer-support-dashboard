# Case Study: HealthTrack Clinic - Optimizing Appointment Scheduling
## Business Issue: The No-Show & Overbooking Challenge

### Company
HealthTrack Clinic (a fictional medical practice)

### Problem
High patient no-show rates and inefficient scheduling leading to wasted doctor time and long patient wait times.

### Patient Impact Example:
Robert arrives 10 minutes early for his 2 PM appointment. Despite his punctuality, he waits 45 minutes because the clinic overbooked and previous appointments ran late. Meanwhile, Dr. Chen has 30-minute gaps in her schedule due to no-shows.

### Root Cause Hypothesis:
The clinic uses fixed 30-minute slots for all appointments regardless of appointment type, and doesn't account for patterns in no-shows or actual procedure durations.

## Project Solution: Appointment Efficiency Analyzer
### Objective
Analyze historical appointment data to identify patterns in no-shows, appointment durations, and peak times to optimize the scheduling system.

### Tools for Beginner Programmers:
- Python, Faker (for patient/appointment data)
- NumPy
- Lovable (for visualization)
- Probability distributions

## Required Data
You need to effectively capture `patient` and `appointment` information, so you will require the following pieces of information:
### Patient
* Patient ID (UUID)
* Age of patient (Random integer between 18 and 80 using `random`)
* No-show status (Boolean, based on patient's `age`)
* Patient primary issue: `Checkup`, `Follow-up`, `Procedure`, `Consultation` (Random array index, randomly selected using `random`'s `choice` method)
* No-show probability (Decimal value based on how frequently the patient no-showed, determined using the `min` method and multiplying the base no show rate by `random.uniform` with a low of `0.8`, a high of `1.2` and a size of `0.3`)

### Appointments
* Appointment ID (UUID, generated by `faker`)
* Date of appointment (Date, generated by `random`)
* Day of the week (Date - Day format, based on the date of appointment)
* Time slot of the appointment: '9:00AM', '10:00AM', '11:00AM', '1:00PM', '2:00PM', '3:00PM', '4:00PM' (Random array index)
* Patient ID (borrowed from `Patient`)
* Patient age (borrowed from `Patient`)
* Appointment type (borrowed from `Patient`)
* Scheduled duration (Integer)
* Actual duration (Integer)
* Show up status (Boolean)
* Duration difference (Integer)

## How to Use the Data
1. Create a function that does the following:
    1. Generates 300 patient profiles
        1. Each patient's age should be generate randomly between 18 and 80
        2. Determine the base no-show rate as `0.15`
        3. If the patient's age is above `60`, multiply the base no-show rate by 70%
        4. Use Numpy's `random` method to determine the gender of the patient indiscriminately, meaning if `random` generates a number greater than `0.5`, multiply the no-show by 120%
        5. Once all of this is evaluate, the `patient` should add new pieces of data to our `patients` table including patient ID, patient name, age, primary issue, and no show probability
            1. The no show probability should be formatted using [Numpy's `uniform` method](https://numpy.org/doc/stable/reference/random/generated/numpy.random.uniform.html) with a low of `0.8` and a high of `1.2` and a size of `0.3`
    2. Creates a way to store 60 days' worth of `appointments`
    3. Determines the day data collection for appoints starts and stores it in a variable
    4. Determines appointment types with properties that store `base duration` and `variability`
        1. `Checkup`: `base duration` = 20, `variability` = 10
        2. `Follow-up`: `base duration` = 15, `variability` = 5
        3. `Procedure`: `base duration` = 45, `variability` = 25
        4. `Consultation`: `base duration` = 30, `variability` = 15
    5. Sets time slot options to '9:00AM', '10:00AM', '11:00AM', '1:00PM', '2:00PM', '3:00PM', '4:00PM'
    6. Generates 60 days of appointments
        1. Determines the current day relative to the start day
        2. Uses the current day to determine the day of the week
            1. If the current day is a weekend day, there should be fewer appointments
        3. Per each time slot:
            1. Generate a random number of appointments set that day
            2. Choose a patient at random from our `patient` array
            3. Set the patient's appointment type
            4. Set the scheduled duration based on the appointment type
            5. Set the actual duration by using Python's `max` method, with the initial value as `10` and the second value with the appointment type's `variability` in mind (i.e, `duration_info['base_duration'] + random.randint(-duration_info['variability'], duration_info['variability']`)
            6. Determine if the patient showed up based on their `no show probability` and a `random` multiplier
            7. Increase the `no show probability` if the appointment occurs in a time slot at '3:00PM' or '4:00pm' ONLY if `random.random` is greater than their `no show probability` * `1.3`
        4. Once all of this has been evaluated, the `appointment` should add new pieces of data to our `appointments` table including the appointment ID, the current date, the day of the week in day format ("Monday", "Tuesday", etc), the time slot, patient ID, patient age, appointment type, scheduled duration, actual duration, show up status, and the difference between the scheduled duration and actual duration
2. Create a dataframe with `Pandas` to create a basic representation of the `appointments` table
3. Use `pandas` to export a `.csv` file to be used in Lovable in the next section
4. Resolve any errors arising in the terminal as you develop your project
